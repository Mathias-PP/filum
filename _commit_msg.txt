fix(db): align schema with models via migration 002

The hand-written 001_initial migration was missing several columns that
the SQLAlchemy models reference. First seed run on Railway surfaced this
on biblio_cards.slug; same gaps exist for platform, signed_at, and four
source columns. Adding them as 002 keeps existing rows intact (table is
empty in prod anyway) and lets ORM queries succeed.

Columns added
- biblio_cards: slug (indexed), platform (default "other"), signed_at
- sources: position (default 0), authors, published_at, is_pivot

Indexes added
- ix_biblio_cards_slug
- ix_biblio_cards_user_status (composite, matches model __table_args__)
